dum

config

By putting a dum entry in your package.json, you can change some settings.

For most projects, this should be unnecessary.

dum {
	opts {
		pug {pretty true}
	}
}

in-dir

"assets" or "src", whichever exists.

This directory is transformed to out-dir.

out-dir

"public" for "assets", or "lib" for "src".

This directory is written to as the output.

Don't keep anything here, because it'll get written over.

port

8000

Port number for dum serve.

Intentionally not 80. Dum is not a production server.

bower

{"in-dir": "bower_components", "out-dir": "lib"}

Paths to use for copying bower components to out-dir.

bower.out-dir is relative to out-dir.

Set to false to disable.

opts

{}

Keys are the names of jstransformers. The name of jstransformer-pug is "pug".

Values are the options for that transform.

sound

true

Whether to play sounds on errors.
