dum

config

By putting a dumConfig entry in your package.json, you can change some settings.

For most projects, this should be unnecessary.

inDir

"assets" or "src", whichever exists.

This directory is transformed to outDir.

outDir

"public" for "assets", or "lib" for "src".

This directory is written to as the output.

Don't keep anything here, because it'll get written over.

port

8000

Port number for dum serve.

Intentionally not 80. Dum is not a production server.

bower

{inDir: "bower_components", outDir: "lib"}

Paths to use for copying bower components to outDir.

bower.outDir is relative to outDir.

Set to false to disable.

opts

{}

Keys are the names of jstransformers. The name of jstransformer-jade is "jade".

Values are the options for that transform.
