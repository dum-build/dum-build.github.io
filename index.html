dum

It's a chore to write html/css/javascript directly. Tools like jade and stylus make this neater. Dum brings all of these tools together.

It's based on node.js so install that but you won't have to write any JavaScript.

simple

Dum doesn't require plugins or build scripts.

It just applies jstransformers list based on file extensions.

If you don't want files/directories sent to public, just start their names with an underscore (_).

Files with no transformer will be copied directly. bower_components are never transformed.

Dum runs a static web server, so you won't need it in production — just copy public to your web server to deploy.

easy

Your package.json a file listing npm dependencies is your configuration.

scripts {start dum serve},
devDependencies {
	dum dum-build/dum,
	jstransformer-jade ,
	jstransformer-stylus 
}

If you need, you can add more options.

Just open a console in your package directory (the one with assets and package.json in it) and run npm install and npm start and look at localhost:8000.

When you change files, dum will update public, so refresh your browser to see the changes.

go!

Start by cloning a project template:

webgit clone https://github.com/dum-build/dum-web-template.git
librarygit clone https://github.com/dum-build/dum-library-template.git

Libraries work the same as web projects but don't serve anything.

If you think your website has a great feature, break it out into a web component and distribute it with bower.
